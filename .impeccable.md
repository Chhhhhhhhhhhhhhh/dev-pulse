## Design Context

### Users
Developers tracking their own productivity. They use dev-pulse daily — starting focus timers, checking tasks, reviewing GitHub activity. Solo developer or small team, using it alongside actual coding work.

### Brand Personality
- **Precise**: A tool, not a product. Does one thing well.
- **Focused**: Eliminates distraction. Everything on screen earns its place.
- **Quiet**: No decoration. Form follows function, nothing more.

3-word persona: Sharp · Restrained · Functional

### Aesthetic Direction
**Refined developer tool** — the visual quality of a well-made instrument, not a SaaS dashboard.

References: Linear (confidence in spacing), iA Writer (focus), vintage audio equipment (tactile precision).
Anti-references: SaaS dashboards with glass morphism, purple-blue gradients, Inter/Roboto everywhere, dark mode with glowing accents, bounce easing, nested cards, side-tab accent borders.

Dark-first — this is a tool used during deep work. Light theme available.

### Design Principles
1. **Fonts carry meaning**: DM Sans for headings (geometric confidence, not overused), system-ui for body (native, invisible), JetBrains Mono for data (precision)
2. **Color is earned**: single teal accent, applied only where it signals state. Everything else is tinted monochrome. No gradients, no glow effects
3. **Borders over shadows**: 1px hairline borders communicate structure. No drop-shadows or box-shadows for elevation
4. **Space creates hierarchy**: tight grouping for related items, generous separation between sections. No card-nesting
5. **Motion means state**: only animate when it tells you what changed. Deceleration curves only — no bounce or elastic easing

### Technical
- OKLCH color space throughout
- Fixed rem type scale (1.25 ratio)
- Font: DM Sans (headings), system-ui (body), JetBrains Mono (data)
- Sidebar navigation with background-highlight active state (no colored border accent)
- Solid surfaces, no backdrops or glass morphism
- No drop-shadow glow effects on any element
